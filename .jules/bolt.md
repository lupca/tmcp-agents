## 2024-05-15 - [Sequential MCP Tool Execution Bottleneck]
**Learning:** Sequential `execute_mcp_tool` calls when fetching related records (like Brand Identity, Customer Profile, and Product) create a significant N+1 query bottleneck. `asyncio.gather` can be used to parallelize these calls to reduce total fetch time from the sum of all requests to the maximum of the individual requests.
**Action:** Always identify independent asynchronous I/O operations (like database or API calls via MCP) and wrap them in an `asyncio.gather` block. Extract data fetching and parsing into a helper coroutine. Ensure `return_exceptions=True` is used to gracefully handle both mandatory and optional fetches, and emit all necessary status SSE events sequentially *before* parallel execution to keep the UI informed.
